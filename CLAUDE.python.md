# CLAUDE.python.md

Короткие инструкции для ИИ при генерации Python-скриптов.

## Общие правила
- Эмодзи: не использовать. Пиши только текст
- Все файлы сохраняй в кодировке UTF-8, все нужные директивы в файле должны быть.
- Методы и функции должны быть максимально типизированны (использовать type hints и Pydantic)
- По возможности добавлять к методам тип возвращаемого значения
- Придерживаться принципов чистого кода (Clean Code) и SOLID
- Избегать избыточной сложности и дублирования кода (DRY)
- Использовать асинхронное программирование там, где это уместно
- Если не указано иное, использовать асинхронные методы и функции (async/await) для операций ввода-вывода и сетевых запросов
- Обновлять документацию метода/класса/модуля (docstrings) при изменении кода
- Не генерировать usage_example_*.py, test_*.py без явного указания
- Многострочные строки: использовать тройные кавычки (""" или '''), НЕ конкатенировать через "+" и "\n"

## Зависимости
- requirements.txt: Внешние библиотеки фиксируй в `requirement.txt`.
- Установка должна работать через: `pip install -r requirement.txt`.

## Логирование
- Используй стандартный модуль `logging`.
- Логи складывай в папку `logs/`.
- Если папки `logs/` нет, создавай её.
- По возможности логируй и в файл, и в консоль.
- **НЕ использовать** `print()` для вывода информации
- **Использовать** `logger.info()` и `logger.debug()`

## Структуры данных
- **НЕ использовать** `dict` для сложных структур данных
- Pydantic: **Использовать** Pydantic `BaseModel` для всех структур данных

## Запуск и интерфейс
- Каждый скрипт должен корректно запускаться без аргументов
- использует argparse
- При запуске без аргументов скрипт печатает справку (help/usage) и завершает работу без ошибки.
- типичные примеры запуска
```bash
# Показывает HELP
python my_script.py 
python my_script.py --help
python my_script.py help

# Выводит список команд, если скрипт с командами.
python my_script list

# Выводит help по команде
python my_script help <command>

# Вызывает команду
python my_script <command> [args...]
```

## Состояние
- tinydb: Если скрипту нужно сохранять состояние между запусками, используй `tinydb`.
- state/: Храни состояние в отдельном файле (например, в папке `state/` или рядом со скриптом).
