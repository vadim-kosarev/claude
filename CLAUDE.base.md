# CLAUDE.base.md

Базовые универсальные инструкции для AI агента.

## Языковые настройки
- **Язык кода:** английский
- **Язык общения:** русский

## Общие правила кода
- **Принцип:** код должен быть самокомментированным для любого языка

## Среда разработки

### Основные параметры
- **IDE:** JetBrains IDEA
- **Shell:** PowerShell (Windows) - команды для run_in_terminal должны быть PowerShell-совместимыми. ОСЬ - Windows.
- для новых терминалов, используемых агентом нужно вызывать команду
   ```chcp 65001 ; $env:PYTHONIOENCODING = 'utf-8'``` - только один раз на терминал, не каждый раз перед командой
- после этого один раз активировать виртуальное окружение из каталога проекта скриптом по полному пути:
   ```GIT_IDE_PROJECT_ROOT\.venv\Scripts\Activate.ps1 2>&1```

### PowerShell команды (важно!)
При использовании `run_in_terminal` tool:
- Разделитель команд: `;` (НЕ `&&`)
- Чтение файлов: `type` (НЕ `cat`)
- Использовать только PowerShell-совместимые команды
- Команды 'head' и 'tail' НЕ использовать, вместо них использовать:
  - `Get-Content -TotalCount n` для чтения первых n строк файла
  - `Get-Content -Tail n` для чтения последних n строк
- **ОБЯЗАТЕЛЬНО** добавлять `[Console]::Out.Flush()` в конце команды для гарантированного вывода результата

**Пример:**
```powershell
cd path; python script.py    # Правильно
cd path && python script.py  # ОШИБКА в PowerShell
```

## Работа с файлами

### Создание файлов
- **НЕ создавать** тестовые скрипты/данные без явного запроса
- **НЕ создавать** `*.md` файлы без явного запроса
- **НЕ дублировать** существующие файлы проекта
- При создании файлов учитывать структуру проекта в IDEA

### Тестовые скрипты
- **НЕ генерировать** `test_*.py` без явного запроса
- **НЕ генерировать** `usage_example_*.py` без явного запроса
- Генерировать тестовые скрипты ТОЛЬКО если пользователь явно запросил

### Документация и отчеты
- если генеришь `SUMMARY_*.md`, `COMPLETE_*.md` и прочие файлы отчетов -- класть их в папку doc в рабочем каталоге
- если генеришь тесты, класть их в папку test в рабочем каталоге, вызывать из рабочего каталога
- **ОБЯЗАТЕЛЬНО обновлять** `README.md` в рабочем каталоге по мере выполнения работы
  - Добавлять описание новых функций/модулей
  - Обновлять инструкции по использованию
  - Документировать важные изменения
  - Актуализировать примеры кода
- Генерировать документацию ТОЛЬКО если пользователь явно попросил

#### Если отчет запрошен явно:
1. **Папка:** `.ai/`
2. **Формат имени:** `YYYYMMDD.NNN_<название>.md`
   - `YYYYMMDD` - текущая дата
   - `NNN` - порядковый номер (трехзначный, с ведущими нулями)
3. **Порядковый номер:**
   - Читается из файла `.ai/version.txt`
   - Инкрементируется при создании нового отчета
   - Сохраняется обратно в `.ai/version.txt`

**Пример:**
```
.ai/20260215.001_api_integration_report.md
.ai/20260215.002_database_migration_report.md
.ai/version.txt  # содержит: 002
```
